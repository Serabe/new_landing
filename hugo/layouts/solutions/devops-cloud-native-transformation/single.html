Vendor Management

With the digital revolution, the number of software vendors has been increasing exponentially. source{d}, helps IT executives monitor, measure and manage vendor adoption and relationships from evaluations to performance and ROI analysis

Examples of questions you should have the answers to:

  • What is the most popular logging library or CI system?
  • What technology vendors should we be engaging with based on actual usage?
  • Have all our teams upgraded to a given vendor product version?

Examples of metrics you could be tracking:

  • Top Vendors by category
  • Fastest Growing / Declining Vendors
  • Share of adoption of key technologies broken down by team

Dependency / Change Management

The adoption of new open source projects, microservices architectures or Cloud Native technologies often comes with potential pitfalls and complexity. source{d} help IT managers assess, execute and manage change and 3rd party dependencies.

Examples of questions you should have the answers to:

  • What external open source libraries are we depending on?
  • What Open Source project should we be contributing back to?
  • What teams are going to be impacted by a given change / releases?

Examples of metrics you could be tracking:

  • Dependency classification breakdown
  • Top internal vs external imports
  • 3rd party imports heatmap

Cloud Migration

Most large enterprises are embracing a hybrid cloud strategy and moving non-critical applications to the Cloud. However, cloud migration can often be more challenging than expected. By extracting insights from your source code and configuration files, source{d} Enterprise helps you achieve your cloud migration objectives faster.

Examples of questions you should have the answers to:

  • What applications can we quickly and safely move to the Cloud?
  • What applications are using Helm vs Docker Compose for deployments?
  • Do applications have an API surface area? Is the Application for internal or external use?

Examples of metrics you could be tracking:

  • Share of repositories with a Dockerfile
  • Share of repositories containing Helm charts
  • Share of API code per repository
